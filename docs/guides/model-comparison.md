# 模型对比

## 综合对比表

| 模型 | 年份 | 输入 | 参数量 | 核心创新 | 激活 | 分类器 |
|------|:---:|:---:|:---:|------|:---:|------|
| LeNet-5 | 1998 | 32×32×1 | ~62K | CNN 开山之作 | Tanh | 2 层 FC |
| AlexNet | 2012 | 224×224×3 | ~58M | ReLU + Dropout + 大卷积核 | ReLU | 3 层 FC (4096) |
| VGG11 | 2015 | 224×224×3 | ~132M | 全 3×3 卷积 | ReLU | 3 层 FC (4096) |
| VGG13 | 2015 | 224×224×3 | ~133M | 同上，更深 | ReLU | 3 层 FC (4096) |
| VGG16 | 2015 | 224×224×3 | ~134M | 同上，更深 | ReLU | 3 层 FC (4096) |
| VGG19 | 2015 | 224×224×3 | ~136M | 同上，最深 | ReLU | 3 层 FC (4096) |
| NiN | 2014 | 32×32×3 | ~1.0M | mlpconv + GAP | ReLU | GAP（无 FC） |
| GoogLeNet | 2015 | 224×224×3 | ~5.3M | Inception + GAP | ReLU | GAP + 1 层 FC |

---

## 参数量 vs 准确率



| 模型 | 参数量 (M) | CIFAR-10 预期准确率 | 效率 (Acc/Params) |
|------|------:|:---:|:---:|
| LeNet-5 | 0.06 | ~70% | 极高（每 M 参数 1000+% 准确率） |
| NiN | 1.0 | ~88% | 极高 |
| GoogLeNet | 5.3 | ~92% | 很高 |
| AlexNet | 58 | ~85% | 低 |
| VGG11 | 132 | ~90% | 很低 |
| VGG16 | 134 | ~93% | 很低 |

**结论**: 参数量与准确率并非线性关系。GoogLeNet 用 5.3M 参数达到接近 VGG16（134M）的水平——结构设计比堆参数更重要。

---

## 推理速度 vs 参数量



| 模型 | 参数量 (M) | ~推理时间 (ms/batch) | FLOPs (G) |
|------|------:|:---:|:---:|
| LeNet-5 | 0.06 | <1 | <0.01 |
| NiN | 1.0 | ~3 | 0.18 |
| GoogLeNet | 5.3 | ~10 | 1.43 |
| AlexNet | 58 | ~15 | 1.81 |
| VGG11 | 132 | ~50 | 13.3 |
| VGG16 | 134 | ~60 | 15.5 |

**结论**: AlexNet 参数量大但 FLOPs 不高（大量参数在 FC 层），推理不算太慢。VGG 最慢——全 3×3 卷积计算密集。

## 架构哲学对比

### 卷积核设计

| 模型 | 核大小 | 设计哲学 |
|------|:---:|------|
| LeNet-5 | 全部 5×5 | 单一尺度 |
| AlexNet | 11×11 → 5×5 → 3×3 | 由大到小，层层递减 |
| VGG | 全部 3×3 | 小核堆叠 = 大感受野 + 省参数 |
| NiN | 5×5 + 1×1 ×2 | 小核 + 逐像素 MLP |
| GoogLeNet | 1×1 / 3×3 / 5×5 并行 | 多尺度同时提取 |

### 分类器设计

| 模型 | 分类器 | FC 参数占比 |
|------|------|:---:|
| LeNet-5 | FC(120→84→10) | 17.8% |
| AlexNet | FC(9216→4096→4096→N) | 93.7% |
| VGG | FC(25088→4096→4096→N) | 89.0% |
| NiN | GAP（无 FC） | 0% |
| GoogLeNet | GAP + FC(1024→N) | 0.2% |

GAP 的引入（NiN, 2014）终结了"大 FC 层主导参数"的时代——后续所有主流架构（ResNet、DenseNet、EfficientNet）均采用 GAP。

## 使用场景推荐

| 场景 | 推荐模型 | 理由 |
|------|---------|------|
| 学习 CNN 基本原理 | LeNet-5 | 结构简单、公式清晰、代码短 |
| 嵌入式 / 移动端部署 | NiN | 1M 参数，无 FC，推理快 |
| 平衡速度与精度 | GoogLeNet | 5M 参数，接近 VGG 精度 |
| 追求最高准确率 | VGG16 | 容量大，但需要 GPU |
| 快速实验 | AlexNet | 中等的参数量和速度 |
| 消融实验（深度影响） | VGG11/13/16/19 | 控制变量：仅深度不同 |

---

## 决策树

```
输入是灰度还是 RGB?
├─ 灰度 → LeNet-5（简单高效）
└─ RGB
   ├─ 输入 < 100×100?
   │  └─ NiN（小输入专用，GAP 自适应）
   └─ 输入 ≥ 224×224
      ├─ 需要极轻量?
      │  └─ GoogLeNet（5M 参数，接近 VGG 精度）
      ├─ 需要简单稳定?
      │  └─ AlexNet（经典架构，好理解）
      └─ 需要最高精度、不在乎计算量?
         └─ VGG16（容量最大）
```
